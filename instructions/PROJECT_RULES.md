# PROJECT_RULES.md

# General principles

Every project should be treated as an architectural narrative.

Projects should communicate:
- context
- process
- methods
- outcomes
- reflections

The portfolio should prioritize quality over quantity.
Fewer well presented projects are preferred over many unfinished ones.

------------------------------------------------

# Required project information

Every project must contain:

- Title
- Subtitle
- Year
- Location (if applicable)
- Project type
- Personal role
- Collaborators (if applicable)
- Software and tools used
- Themes
- Project description
- Images


------------------------------------------------

# Project structure

Projects should follow the following structure:

1. Introduction

A short project description.


2. Context

Explain:
- why the project exists
- the design challenge
- the project's ambitions


3. Methods

Examples:
- architectural design
- GIS analysis
- workshops
- research methods
- parametric design
- AI workflows
- sustainability analysis


4. Process

Show:
- sketches
- diagrams
- drawings
- iterations
- analytical work


5. Outcomes

Present:
- final proposals
- plans
- sections
- renders
- visualizations


6. Reflection

Whenever possible explain:

- what was learned
- challenges encountered
- limitations
- future opportunities


------------------------------------------------

# Themes

Projects may belong to multiple themes.

Examples:

- Architecture
- Urbanism
- Research
- Emerging Technologies
- Sustainability
- Transformation
- Housing
- Landscape
- Computational Design


------------------------------------------------

# Images

Prioritize:

- architectural drawings
- diagrams
- analytical material
- process images

Avoid:

- decorative imagery
- excessive rendering
- duplicated images.


------------------------------------------------

# Writing principles

Writing should be:

- analytical
- concise
- thoughtful
- accessible
- research driven


Avoid:

- marketing language
- unnecessary buzzwords
- exaggerated claims.


------------------------------------------------

# Future adaptability

Projects should always be reusable.

Claude may:

- reorder projects
- highlight projects
- group projects differently

Claude may never:

- remove project content
- delete projects
- modify project descriptions without approval.


------------------------------------------------

# General principle

Architecture is more than the final image.

Whenever possible prioritize:
- context
- process
- methods
- reflection

over aesthetics alone.
